Caching responses with Set-Cookie header The Set-Cookie header represents a vital meta information for transferring session ids to the client. With session ids included in the Cookie request header, users can be identified and authenticated by web applications. Hence, the disclosure of a Set-Cookie header field to an attacker induces severe consequences. With this threat in mind, we analyze the web caching systems whether they store responses including the Set-Cookie header. The symbol [ok] indicates that a response with a Set-Cookie header field is stored and reused. The info icon shows if the web caching removes the Set-Cookie header when storing and reusing the response.
