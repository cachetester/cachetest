Caching responses with Set-Cookie header The Set-Cookie header represents a vital meta information for transferring a session id to the client. With this session id included in the Cookie header field of the requests, users can be identified and authenticated by web applications. Hence, the disclosure of a Set-Cookie header field to an attacker induces severe consequences. With this threat in mind, we analyze the web caching systems whether they store responses including a Set-Cookie header field.
