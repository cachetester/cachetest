Client-originated policies Clients can define cache-related requirements for response as well. To do so they can use the control directives max-age, max-stale, no-store, min-fresh, only-if-cached and no-cache. This parameter must be included in the Cache-Control header. max-age, no-store and no-cache has same semantic meaning as the same control directives for the response. With max-stale clients declares to accept a response with a maximum staleness. The min-fresh parameter requires that the returned response must still be fresh for at least the specified seconds. Client sending only-if-cached are only willing to receive a stored response. If the cache does not have a suitable cache one, the web caching system must return a 504 Gateway Time-out. The [ok] indicates the caches behaves according the requirements defined for corresponding control directive. The [fail] symbol declares that the client-side control directive is ignored by the cache.
