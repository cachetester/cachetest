Explicit freshness lifetime This Table shows if the proxy caches honor the freshness lifetime definition of the control directives, max-age, s-maxage and the Expires header. The max-age and s-maxage control directives defines the relative freshness lifetime in seconds. The only difference between max-age and s-maxage is that s-maxage only applies for shared caches. The max-age parameter is dedicated for all caches, i.e. private and shared caches. The symbol [ok] indicates that this control directive or header field is supported properly. The disobey of freshness lifetime definitions is shown by the symbol [fail]. Also, the Table depicts if the proxy caches comply with the order of priority. The RFC 7234 requires to prefer max-age when the max-age control directive and the Expires header are present simultaneously. If max-age and s-maxage are set at the same time, a shared cache must consider s-maxage while a private cache must take account of max-age. The symbol [ok] depicts also the compliance with the order of priority while [fail] indicates a non-conformance. Tests results which are non-determinable are declared with [x].
