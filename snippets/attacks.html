Further security issues Security consideration on web caching attacks are described in RFC 7230 at Section 9. This table shows if the web caching systems comply with security requirements defined in web caching standard. The analysis focus on two attacks:
  1. HTTP request smuggling which results from parsing two Content-Length header, even though it is prohibited
  2. Web deception attack that exploits the misbehavior that implicit freshness lifetime policies are preferred over explicit caching requirements.
The symbol [ok] indicates that the web caching systems honor the security requirements and [fail] implies a dishonor.
